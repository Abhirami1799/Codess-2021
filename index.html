About

The Grid:
A graph, with each cell representing a node. Each cell is connected to adjacent cells horizontally, vertically, and diagonally by invisible edges that (by default) cost to traverse.
Click on Start Node and End Node buttons to move around the Source and Destination nodes respectively.

Weights/Walls:
Drag or click anywhere on the grid to place or remove weights or walls.
- Weighted cells: These cells have edges that cost 5 to traverse horizontally or vertically. They represent environments that are difficult to cross.
(Note : Breadth First Search is an unweighted algorithm.)
- Walled cells: These cells do not have edges connecting them to adjacent cells. They represent obstacles.

Speed Selection:
Use the dropdown to toggle between simulation speeds for two node path-finders.

Station Node:
CLick anywhere to add multiple stations. The Traveling Salesman problem finds the least cost path that starts from Source (green), covers all Stations (blue) and the Destination (red),
and comes back to Source. The Multiple Stops algorithm is a path finder that attempts to find the optimum path from Source to Destination covering as many stations in between as possible,
in order that the cost incurred is lesser than or equal to the maximum cost given by the user (default : 120).
Feel free to experiment and don't forget to have fun!
